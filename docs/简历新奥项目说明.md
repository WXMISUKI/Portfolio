# 简历项目说明

## 项目概述

**项目名称**：新奥海底管道预警系统（xinao-gis-screen）

**项目周期**：2025年12月 - 2026年4月

**项目角色**：主要负责人

**项目简介**：基于 GIS 的海底管道预警监控平台，用于实时监控舟山海域船舶航行，当船舶接近海底管道电子围栏时自动发出预警并支持驱离操作。系统集成了实时船舶监控、智能预警、视频监控、VHF 广播等功能，为海底管道安全提供全方位保障。

## 核心功能模块

### 1. 实时船舶监控
- 通过 SSE 实时获取船舶 AIS 数据，支持 1000+ 艘船舶同时监控
- 船舶信息包括：MMSI、船名、经纬度、航向、航速、船首向等
- 船舶图标根据船型和国籍自动映射显示
- 支持船舶筛选、搜索、定位等操作

### 2. 电子围栏管理
- 支持自定义绘制电子围栏（多边形、圆形）
- 锚地电子围栏管理，支持分组显示
- 围栏与图层控制关联，支持显示/隐藏切换
- 围栏编辑模式和删除功能

### 3. 智能预警系统
- **三级预警体系**：根据船舶与管道距离自动触发不同等级预警
  - 注意（黄色）：2000-5000m
  - 警告（橙色）：500-2000m
  - 危险（红色）：< 500m
- **动态预警配置**：支持从后端动态获取管道预警范围配置
- **端点渐变效果**：管道预警范围两端使用线性渐变，视觉效果更柔和
- **实时预警通知**：预警弹窗在屏幕右侧垂直居中显示，最多展示 3 个预警
- **告警状态管理**：支持告警忽略、结束、闭环等操作
- **多客户端同步**：通过轮询机制实现多客户端告警状态同步

### 4. 视频监控
- **多格式支持**：支持 RTSP、FLV、HLS、MJPEG 等多种视频流格式
- **RTSP 转 FLV**：基于 SRS + FFmpeg 实现低延迟视频流转换（1-2秒）
- **PTZ 云台控制**：支持云台上下左右移动、变焦、预置位等操作
- **视频截图和录制**：支持手动抓拍和视频录制功能
- **视频跟踪**：支持通过经纬度参数进行视频跟踪联动
- **视频资源管理**：支持摄像头列表管理和状态监控

### 5. VHF 广播系统
- VHF 登录功能，支持获取和管理 VHF token
- 语音播报功能，支持文本转语音
- 通讯录管理，支持联系人分组和搜索
- 广播消息发送，支持圈选船舶批量发送
- 广播历史记录查询

### 6. 船舶历史轨迹回放
- 支持查询船舶历史轨迹
- 轨迹播放功能，支持播放速度调整
- 轨迹时间范围选择
- 轨迹点高亮和船舶图标显示

### 7. 船舶筛选和白名单管理
- 船舶筛选功能，支持按船型、国籍等条件筛选
- 白名单管理，支持导入/导出白名单船舶
- 白名单船舶自动跳过高亮
- 分页显示，支持自定义每页数量

### 8. 地图工具
- **海图切换**：支持亮色、暗色、卫星、海图等多种底图切换
- **天地图集成**：支持天地图瓦片加载
- **距离量算**：支持地图上两点间距离测量
- **面积量算**：支持多边形面积计算

## 技术难点与解决方案

### 1. RTSP 转 FLV 视频流转换

**问题描述**：需要将 RTSP 视频流转换为浏览器可播放的 HTTP-FLV 格式，实现低延迟视频监控。

**解决方案**：
- 基于 SRS（Simple Realtime Server）和 FFmpeg 构建视频流转换模块
- 使用 FFmpeg 将 RTSP 流转换为 RTMP 格式推送到 SRS 服务器
- SRS 服务器接收 RTMP 流并提供 HTTP-FLV 输出
- 前端使用 flv.js 播放器实现低延迟播放（1-2秒）
- 统一流名规则，直接使用通道号作为流名，简洁明了
- 支持多环境配置（开发、测试、生产）
- 提供健康检查脚本和 Docker 部署方案

**技术要点**：
- 使用 `-c:v copy` 避免重新编码，降低 CPU 占用
- 使用 TCP 传输 RTSP（`-rtsp_transport tcp`）提高稳定性
- 启用低延迟模式，使用 `-fflags nobuffer -flags low_delay` 参数
- 流名规则：直接使用通道号（如 101、201），无需额外前缀

### 2. HLS 视频流代理配置

**问题描述**：HTTPS 页面加载 HTTP HLS 视频流时出现 Mixed Content 错误，且大华平台返回的视频流 URL 无法正常访问。

**解决方案**：
- 通过 Nginx 代理将 HTTPS 请求转发到 HTTP 视频流服务器
- 配置 `/ext-video` 代理路径，解决 Mixed Content 问题
- 正确配置代理目标地址，直接使用大华平台返回的端口 83
- 添加 CORS 支持和 Cookie 传递
- 禁用 Gzip 防止视频流乱码

**关键配置**：
```nginx
location /ext-video/ {
    proxy_pass http://172.16.16.2:83/;
    proxy_ssl_verify off;
    gzip off;
    add_header Access-Control-Allow-Origin $cors_origin always;
}
```

### 3. FLV WebSocket 规范问题

**问题描述**：WebSocket FLV 流格式不正确，导致 flv.js 无法播放。

**解决方案**：
- 确保 FLV 头是第一个发送的数据包（13字节）
- 如果流中没有音频，将 Flags 设为 0x01 而不是 0x05
- 正确的发送顺序：FLV头 → Script Tag → Video Sequence Header → Video NALU
- 使用 FFmpeg 标准参数生成 FLV 流：`-c:v copy -an -f flv`

### 4. HLS 播放器优化

**问题描述**：HLS 视频播放出现 DEMUXER_ERROR_COULD_NOT_PARSE 错误，分片文件 404，播放闪现/卡顿/跳播。

**解决方案**：
- 修正浏览器 HLS 支持判断逻辑，只有 `'probably'` 才使用原生 HLS
- 显式设置 `xhr.withCredentials = true` 确保 Cookie 传递
- 移除 hls.js 配置冲突，统一使用基于分片数量的配置
- 彻底移除业务侧追加的 `_t=` 时间戳参数，使用稳定 URL
- 优化加载状态管理，在 `MANIFEST_PARSED` 时立即清除 loading
- 优化错误处理，分片 404 时不标记为 fatal
- 关闭低延迟模式，减少不必要的刷新

### 5. 大数据量船舶加载优化

**问题描述**：每次刷新需要加载 1000+ 艘船只，页面卡顿 3-5 分钟，完全无法拖动。

**解决方案**：
- **Pinia Store + LocalStorage 双层缓存**：避免重复 API 请求，5 分钟缓存有效期
- **一次性加载 + 渐进式UI反馈**：所有数据一次性加载到地图，使用 requestAnimationFrame 优化渲染
- **Canvas 高性能渲染**：使用 OpenLayers ImageLayer + Canvas 2D 实现高性能船舶渲染
- **视口裁剪**：仅渲染当前视口范围内的船舶，避免无效渲染
- **层级渲染**：根据缩放级别自动切换渲染样式（圆点/三角形/带标签）
- **标签防重叠**：智能标签布局算法，自动避免标签重叠
- **图层顺序优化**：确保船只图层始终在最上层，不会被电子围栏覆盖

**性能提升**：
- 首屏加载时间：3-5 分钟 → 2-3 秒（95%↓）
- 刷新页面时间（5分钟内）：3-5 分钟 → < 100ms（99%↓）
- UI 响应性：完全卡死 → 立即可拖动（100%↑）
- API 请求次数（5分钟内）：每次刷新 → 仅1次（90%↓）

### 6. 多客户端告警状态同步

**问题描述**：多台电脑同时运行系统时，一台电脑处理告警后，其他电脑的告警列表仍然存在，导致数据不一致。

**解决方案**：
- **前端轮询兜底方案**：每隔 10 秒调用接口检查当前队列中的告警是否已被处理
- **增量更新**：只检查本地队列中存在的告警，避免无效查询
- **自动同步**：如果发现状态变化（已处理/已闭环），自动从本地列表中移除
- **状态标记**：使用 `handledAlertIds` 集合避免重复处理
- **按 MMSI 分组查询**：减少查询次数，优化性能

**同步延迟**：< 10 秒

### 7. 管道预警动态配置和端点渐变效果

**问题描述**：管道预警范围需要从后端动态获取配置，且预警范围两端需要平滑过渡效果。

**解决方案**：
- **动态预警配置**：从后端 API `/pipeline-coordinate/with-alert-config` 获取管道预警配置
- **分段管理**：使用 `pipelineName + pipelineType` 组合作为唯一键，支持为每个管道分段设置独立的预警配置
- **端点线性渐变**：使用 Canvas 自定义渲染实现管道预警范围两端的线性渐变效果
  - 起点区域（0% 位置）：完全透明（opacity = 0）
  - 渐变区域（0-15% 位置）：透明度从 0 逐渐增加到正常值
  - 中间区域（15-85% 位置）：保持正常透明度
  - 渐变区域（85-100% 位置）：透明度从正常值逐渐降至 0
  - 终点区域（100% 位置）：完全透明（opacity = 0）
- **配置缓存**：配置加载后缓存，避免重复请求
- **降级方案**：配置加载失败时使用环境变量中的默认配置
