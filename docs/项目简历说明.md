# 渔港安全生产监控平台 - 项目简历说明

## 项目概述

**项目名称**：渔港安全生产监控平台（yy-vue）  
**项目类型**：智慧港口/海洋渔业安全生产管理系统  
**技术架构**：Vue 3 + TypeScript + Element Plus + Vite  
**项目周期**：2025年8月 - 至今（持续开发中）

该项目是一个面向渔港安全生产的综合性监控管理平台，集成了船舶实时监控、预警管理、智能管控、应急救援等核心功能，通过物联网、大数据和人工智能技术，实现对渔港船舶、人员、设备的全方位监管，提升渔港安全生产管理水平。

## 主要功能模块

### 1. 船舶 AIS 实时监控系统
- **核心功能**：通过 WebSocket 接收船舶实时位置、航向、航速等数据
- **可视化展示**：在天地图上实时显示船舶图标和雷达目标
- **数据统计**：实时统计船舶数量、雷达数量、船舶类型分布
- **技术亮点**：
  - 每秒更新一次船舶位置
  - 智能雷达目标生成（每5次船舶更新刷新1次）
  - 自动重连机制（最多5次重连尝试）
  - 心跳检测保持连接稳定

### 2. 预警管理系统
- **预警类型**：船舶预警、人员预警、设备预警、温度预警、工作区预警
- **预警流程**：预警管理 → 预警流转 → 预警记录
- **功能特点**：
  - 多维度预警规则配置
  - 预警数据自动筛选和分类
  - 预警详情实时查看
  - 预警状态跟踪和流转

### 3. 智能管控子系统
- **人员管理**：船员档案、人员档案管理
- **船舶管理**：船舶档案、船舶状态监测、船舶管控
- **设备管理**：设备健康管理、设备档案
- **泊位管理**：泊位状态监控、泊位分配
- **渔港安防**：安防监控、预警信息窗口

### 4. 档案管理系统
- **船舶档案**：船舶基本信息、船舶类型、船舶状态
- **船员档案**：人员基本信息、资质证书、年龄与出生年月自动计算
- **船企档案**：企业基本信息、企业资质、企业状态
- **证书管理**：证书信息、证书到期提醒

### 5. 应急管理系统
- **应急预案**：预案管理、预案详情查看
- **救援物资**：物资管理、物资统计
- **救援队伍**：队伍管理、队伍配置

### 6. 海上救援子系统
- **近海事件管理**：事件记录、事件编辑
- **现场还原**：事故现场可视化还原
- **救援数据分析**：救援数据统计和分析

### 7. 历史轨迹分析
- **船舶轨迹回放**：历史轨迹查看和回放
- **轨迹分析工具**：轨迹数据分析和可视化
- **船舶画像**：船舶档案详情查看

### 8. 基础功能模块
- **用户权限管理**：用户、角色、部门、岗位、菜单管理
- **数据字典**：字典类型、字典数据管理
- **参数配置**：系统参数动态配置
- **日志管理**：操作日志、登录日志
- **文件管理**：文件上传、下载、删除
- **定时任务**：任务管理、日志管理、执行器管理

## 解决的关键技术问题

### 1. WebSocket 实时数据性能优化
**问题**：船舶数据每秒更新，频繁重绘导致页面卡顿
**解决方案**：
- 采用增量更新策略，只更新变化的船舶
- 雷达目标每5次船舶更新才刷新1次，减少重绘次数
- 禁用位置更新动画，确保即时响应
- 结果：实现了流畅的实时监控体验，支持100+船舶同时显示

### 2. 组件数据传递和校验问题
**问题**：LocationPicker 组件无法第一时间告知父级数据变化，导致校验错误提示不准确
**解决方案**：
- 使用 ref 正确绑定组件引用
- 优化组件的数据更新回调机制
- 修改校验逻辑，确保数据变化能够及时反馈
- 结果：提升了表单校验的准确性和用户体验

### 3. 页面布局流"冒泡"问题
**问题**：分析工具组件布局流向上传递，影响父元素尺寸导致父级变化
**解决方案**：
- 使用脱离布局流的方式实现组件定位
- 优化组件的 CSS 样式和定位策略
- 结果：解决了布局冲突，提升了页面稳定性

### 4. 大数据量渲染性能优化
**问题**：预警管理系统数据量大，查询和渲染性能不佳
**解决方案**：
- 优化数据筛选逻辑，减少不必要的数据加载
- 使用虚拟滚动技术，提升表格渲染性能
- 实现数据分页和懒加载
- 结果：提升了系统响应速度，支持万级数据量

### 5. 组件复用和代码优化
**问题**：多个模块存在重复代码，维护成本高
**解决方案**：
- 提取公共组件和工具函数
- 开发可复用的校验逻辑（useUniquenessValidator）
- 封装 CRUD 操作的通用逻辑（useCrud）
- 结果：减少了代码重复，提升了开发效率和代码质量

### 6. 数据格式映射和兼容性
**问题**：后端数据格式与前端不一致，导致数据更新失败
**解决方案**：
- 实现统一的数据格式映射函数
- 完善数据校验和转换逻辑
- 优化错误处理和友好提示
- 结果：提升了系统的健壮性和用户体验
