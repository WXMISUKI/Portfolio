## 🐛 第三阶段：主题切换问题修复

### 3.1 问题一：主题切换按钮点击无效果

**用户反馈**:
> "刚刚经过我的测试我发现如下问题。1首先是深色浅色按钮点击后无效果，默认是浅色状态"

**原因分析**:
- CSS 变量未正确导入
- variables.css 中的 CSS 变量未被加载

**解决方案**:
在 `src/assets/styles/main.css` 顶部添加导入：
```css
@import './variables.css';
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### 3.2 问题二：只有头部背景切换，其他部分无变化

**用户反馈**:
> "刚刚经过我的测试，关于上述你增加的主题色切换功能，我发现点击后只有网页的header部分切换了背景色，其他均无变化"

**原因分析**:
- `App.vue` 使用了 `<style scoped>`，阻止了主题类名的作用范围
- CSS 变量作用域受限

**解决方案**:
在 `src/App.vue` 中移除 `scoped` 属性：
```vue
<style> <!-- 原来是 <style scoped> -->
.app {
  min-height: 100vh;
  background-color: var(--color-background);
  color: var(--color-text);
  transition: background-color var(--duration-normal) var(--ease-in-out),
              color var(--duration-normal) var(--ease-in-out);
}
</style>
```

### 3.3 问题三：Home.vue 标签颜色不响应主题切换

**原因分析**:
- Home.vue 中使用了硬编码的颜色值 `rgba(6, 182, 212, 0.1)`
- 未使用 CSS 变量

**解决方案**:
替换为 CSS 变量和主题特定选择器：
```css
html.dark .hero-tags .tag {
  background: rgba(6, 182, 212, 0.1);
}

html.light .hero-tags .tag {
  background: linear-gradient(135deg, rgba(99, 102, 241, 0.1), rgba(139, 92, 246, 0.1));
}
```

### 3.4 问题四：Footer 背景色与主体相同

**用户反馈**:
> "我看苹果官网的样式设计时，其整体分为头部，主体和底部，其底部背景色是#F5F5F7，有和主体做明显区分，我们当前没有对应做好区分"

**原因分析**:
- 没有专门的 Footer 背景色变量
- Footer 使用了与主体相同的背景色

**解决方案**:
在 `src/assets/styles/variables.css` 中添加 Footer 变量：
```css
:root {
  --color-footer-bg: #0F172A;
  --color-footer-border: #1E293B;
}

:root.light {
  --color-footer-bg: #FFFFFF;
  --color-footer-border: #E5E7EB;
}
```

### 主题切换实现

**useTheme Composable**:
```typescript
export function useTheme() {
  const theme = ref<Theme>(getStoredTheme())

  const setTheme = (newTheme: Theme) => {
    theme.value = newTheme
    document.documentElement.classList.remove('light', 'dark')
    document.documentElement.classList.add(newTheme)
    localStorage.setItem('theme', newTheme)
  }

  const toggleTheme = () => {
    setTheme(theme.value === 'dark' ? 'light' : 'dark')
  }

  onMounted(() => {
    if (!getStoredTheme()) {
      const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
      setTheme(prefersDark ? 'dark' : 'light')
    }
  })

  return {
    theme,
    setTheme,
    toggleTheme
  }
}
```

## 💡 关键决策记录

### 1. 主题切换方案

**决策**: 使用 CSS 变量 + HTML class 切换

**理由**:
- 性能优秀
- 易于维护
- 浏览器支持好
- 与 Vue 3 兼容

**替代方案**: CSS-in-JS (Tailwind darkMode)
**拒绝理由**: 需要大量重构，学习成本高

### 2. 组件库开发策略

**决策**: 先 UI 组件，后业务组件

**理由**:
- UI 组件是基础
- 业务组件依赖 UI 组件
- 遵循从底向上的开发原则

### 3. 页面完善顺序

**决策**: 项目经验 → 首页 → 关于我 → 技术栈 → 联系我

**理由**:
- 项目经验是核心展示内容
- 有完整的组件支持
- 数据结构清晰
- 可以立即看到效果

### 4. 截图方案

**决策**: 先用占位图，后替换为真实截图

**理由**:
- 可以立即开始开发
- 不阻塞开发进度
- 用户可以并行准备截图
